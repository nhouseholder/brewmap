## What Was Done

### Frontend Modularization
- Extracted 300 lines of inline CSS from `public/index.html` to `public/styles.css`
- Extracted ~900 lines of inline JavaScript into 10 ES modules in `public/js/`:
  - `config.js` ‚Äî constants (FLAVOR_TAGS, FLAVOR_BAR_COLORS, CITIES)
  - `utils.js` ‚Äî pure utilities (hashCode, mulberry32, getDistance, escHtml, etc.)
  - `state.js` ‚Äî centralized mutable state with explicit setters
  - `map.js` ‚Äî Leaflet map init, user marker, map markers
  - `render.js` ‚Äî all DOM rendering (shop list, detail panel, featured rails, stars, etc.)
  - `filters.js` ‚Äî filter logic (flavor filters, roast filters, sorting, filterShops)
  - `data.js` ‚Äî data pipeline (loadCachedCities, loadFromCache, scanArea, review generation)
  - `city-search.js` ‚Äî city search, switching, geolocation services
  - `mobile-sheet.js` ‚Äî mobile bottom sheet touch handling
  - `app.js` ‚Äî bootstrap, event wiring, window exposure for inline HTML
- Rewrote `public/index.html` as a thin shell with no inline CSS or JS
- Replaced inline event handlers with data attributes + event listeners where possible
- Updated version badge to `v6.3.0 / Apr 24, 2026`

### Backend Unification
- Created `shared/overpass.mjs` and `shared/flavors.mjs` as canonical sources of truth
- Updated `scripts/lib/overpass.mjs`, `scripts/lib/flavors.mjs`, `functions/lib/overpass.mjs`, `functions/lib/flavors.mjs` to re-export from shared
- Eliminated the silent fork between Node.js and Workers versions

### API Hardening
- Refactored `functions/api/city/discover.js`:
  - Split handler (validation, rate limiting, response formatting) from service (business logic)
  - Added structured error responses with codes: `INVALID_INPUT`, `RATE_LIMITED`, `OVERPASS_UNAVAILABLE`, `INTERNAL_ERROR`
  - Added per-IP rate limiting via `CF-Connecting-IP` header
  - Removed `updateCitiesIndex()` to eliminate the read-modify-write race condition on the cities index
  - Newly-discovered cities are accessible by direct slug but won't appear in `/api/cities` until the next weekly harvest
